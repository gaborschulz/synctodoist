TodoistAPI

TodoistAPI

You can initialize TodoistAPI in two ways: either with a Settings object in the settings argument, or you can provide your settings as arguments to the initializer. These will be passed on directly to the initializer of the Settings object.

The Settings model will try to infer your settings from environment variables, and you can configure it to use Docker secrets inside containers. For more details, please, check the documentation of the Settings model.

Examples:

>>> from synctodoist import TodoistAPI
>>> api = TodoistAPI()

or

>>> from synctodoist import TodoistAPI
>>> from synctodoist.models import Settings
>>> settings = Settings(...)
>>> api = TodoistAPI(settings=settings)

or

>>> from synctodoist import TodoistAPI
>>> api = TodoistAPI(api_key="...", cache_dir="...")

Parameters:

Name Type Description Default
settings Settings | None

an instance of the Settings class

 None
**kwargs

keyword arguments that should be passed on to the Settings object. These will be passed on only if the settings argument is not provided.

commit()

Commit open commands to Todoist.

Commands are processed in batches to be frugal with the request limits defined by the Todoist API (check Limits in the Todoist developer documentation for more details).

Examples:

>>> from synctodoist import TodoistAPI
>>> api = TodoistAPI()
>>> api.commit()

Raises:

Type Description
TodoistError

if the Todoist Sync API responds with an error to your request.

move_task(task, parent=None, section=None, project=None)

Move task to a different parent, section or project

Important

One of the parameters has to be provided.

To move an item from a section to no section, just use the project_id parameter, with the project it currently belongs to as a value.

Parameters:

Name Type Description Default

 required

 None

 None
project str | int | Project | None

the project in which you want to place the task

 None

Raises:

Type Description
